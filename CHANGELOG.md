# 更新日志

## v2.8.1

### 新功能



### 修复

- **修复弹幕源自动更新导致的无限重启循环** — 逐文件/增量自动更新时，新版本因备份调用参数不匹配而从未真正持久化到备份目录（scrapers 目录不持久化，重启后回退旧版），造成"下载新版→重启→回退→再下载"的无限重启循环。现修复：① 备份接口支持写入新版本信息到持久化备份目录；② 自动更新改为基于"版本状态"决策——仅在确认新版本成功落盘备份目录后才重启，且当备份目录已是目标版本（上一轮已下载好、只差重启生效）时不再重复下载，直接触发重启，从根本上切断重启循环。
- **修复上传弹幕源离线包后无法生效** — 上传离线包后自动重启容器时因错误的模块导入（`get_config_manager`）崩溃，导致新版本卡在备份目录无法加载；同时上传写入备份的版本信息未刷新更新时间，重启后可能不被识别为更新。现修复导入错误，并在上传时刷新备份版本时间戳，确保重启后新版本被正确恢复生效。
- **修复单集弹幕反复触发自动刷新的死循环** — 当某分集从源站抓到的弹幕数不比库内更多时，刷新会跳过写入且不更新"最后刷新时间"，导致自动刷新的"距今超过阈值"判断永远成立，同一集被反复触发刷新、后台任务持续空转。现修复：单集刷新与批量刷新在"抓取成功但弹幕未新增"时也更新最后刷新时间戳（完全未抓到弹幕的情况仍保留重试机会）。
- **修复后备匹配超时时间配置不生效** — `/match` 接口等待后备匹配的超时时长此前硬编码为 30 秒，导致 WebUI"后备匹配超时时间"（matchFallbackTimeout）配置无法生效；现已改为读取该配置（默认 60 秒，设为 -1 表示无限等待直到匹配完成），超时后返回未匹配，匹配任务继续在后台运行。
- **修复媒体库"导入整季/删除整季"按钮点击无反应** — 季度按钮此前通过 key 反推父剧节点定位标题，因 key 命名规则不匹配导致定位失败而静默失效；现改为直接使用季度节点自带的剧名与服务器信息，导入/删除整季恢复正常（勾选后批量导入不受影响）。
- **修复手动刷新分集后不再发送通知** — WebUI 单集刷新与批量刷新提交任务时未携带任务标识，导致完成后通知事件类型判定为空而不发送任何通知；现已补齐标识，刷新完成会正确归类为"刷新成功/失败"通知（是否推送仍取决于通知设置中对应事件的订阅）。
- **修复获取弹幕接口偶发 500（NameError）** — `/api/v2/comment/{episodeId}` 的自动刷新检测块存在缩进错误：`unique_key` 仅在弹幕"已过期"分支中赋值，但后续"检查任务/提交刷新/等待重取"代码却跳出了该判断始终执行；当分集已有弹幕且尚未到自动刷新天数（未过期）时，会因引用未定义的 `unique_key` 抛出 `NameError`，经 anyio TaskGroup 包装为 500 返回。现将该整块逻辑正确收敛到"已过期"分支内，未过期分集恢复正常返回弹幕。

### 优化

- **优化弹幕源搜索的别名分发策略** — 此前搜索会把作品的全部多语言别名（简中/繁中/日文/英文）逐个喂给每个弹幕源，导致单个源反复网络搜索、耗时叠加且常返回 0 结果。现改为各源自行决定搜索关键词：默认只用主搜索词；巴哈姆特动画疯（gamer，繁中站）因简中原词命中率低，特化为从别名中按「中文 > 日文 > 主搜索词」挑选最多 3 个可信别名搜索。新增通用别名语言识别工具（src/utils/alias_language.py）供各源复用。
- **统一后备任务通知样式并补充海报** — 后备匹配完成通知新增作品海报图片；预下载完成通知改为与匹配后备一致的结构化样式（作品名 / 季集 / 弹幕源 + 海报），顶部任务类型标题保持不变（海报仅在支持图片的渠道生效，失败自动降级为纯文字）。
- **搜索结果编辑导入支持第 0 季** — 编辑导入弹窗的季度输入放开最小值限制，允许填入第 0 季（特别篇/SP 等），与媒体库编辑器保持一致。

### 维护

- **消除数据库结构同步的类型误报** — PostgreSQL 中 `DECIMAL` 与 `NUMERIC` 完全等价，此前结构自动同步会对 external_calendar_item 的 rating/platform_rating 字段反复告警"类型不匹配"；现补充 numeric↔decimal 等价规则，不再误报，字段无需手动处理。
