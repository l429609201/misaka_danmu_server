# 更新日志

## v2.8.2

### 新功能

- **新增通知智能洪峰汇总**：批量导入、刷新等成功通知在短时间大量产生时自动聚合，避免消息轰炸；失败通知仍逐条发送。
- **新增 AniBT 元数据与私有 RSS 订阅支持**：支持搜索 AniBT 元数据，并通过私有 RSS 自动同步追番订阅。
- **新增编辑导入年份设置**：可手动填写或修改作品年份，用于区分同名同季的不同作品。
- **新增编辑导入双列表管理**：分集拆分为“待导入”和“不导入”，支持过滤后恢复、拖拽排序及重新编号。
- **新增本机登录锁定清除能力**：本机维护脚本可通过回环地址免登录清除登录失败锁定，远程请求仍需鉴权。
- **企业微信通知支持主动上传海报**：服务端读取图片并上传为企业微信素材，降低外链防盗链或内网地址导致的海报加载失败。

### 修复

- **修复同名作品导入误合并与分裂问题**：弱标题匹配统一按标题、季度、媒体类型和年份状态判断；未知年份仅复用未知年份条目，强标识命中后可安全补齐年份。
- **修复并发导入重复创建作品**：作品身份查找、创建及强标识落库改为原子化串行处理，避免立即任务与下载、后备队列同时创建重复条目。
- **修复不同作品因重复或空 mediaId 导致海报、类型串台**：数据源及元数据命中增加标题一致性校验，空 mediaId 不再参与强标识匹配。
- **修复清空缓存后无法重新触发后备弹幕下载**：可根据 episodeId 编码从数据库反查作品与弹幕源并重建后备信息。
- **修复搜索空结果被缓存三小时**：瞬时超时、限流产生的空分页结果不再写入长时间缓存。
- **修复媒体库失败项目无法再次一键导入**：未导入状态改为依据弹幕库中是否存在真实弹幕判断，不再依赖已提交任务标志。
- **修复弹幕源更新后用户排序丢失**：保存设置时记录顺序快照，启动或重载后自动恢复。
- **修复弹幕源全量替换更新后版本回退及重复重启**：统一版本时间字段，并在重启前校验持久化备份版本。
- **修复媒体库和 Webhook 导入任务重启后无法恢复**：补齐任务类型和恢复参数，使中断任务可重新构建执行。
- **修复数据库连接终止异常刷屏**：HTTP 中间件改为纯 ASGI 实现，SSE 鉴权不再长期占用数据库会话，并过滤无害的连接池关闭噪音。
- **修复编辑导入重复集号互相覆盖**：提交前检测重复编号，并提供一键重新排序。

### 优化

- **优化编辑导入弹窗布局与拖拽体验**：适配桌面端和移动端，修复拖拽覆盖层偏移、分页碰撞及弹窗高度问题。
- **优化分集过滤可见性**：被全局、单剧或源规则过滤的分集进入“不导入”列表，并显示过滤原因，用户可手动恢复。
- **优化应用生命周期与主入口结构**：启动、关闭、控制 API 文档和环境检测拆分为独立模块，精简主程序入口。
- **优化任务和通知信息**：导入、刷新、后备任务补充作品、季度、集数、弹幕源及海报等结构化信息。
- **优化图片读取能力**：统一支持远程 URL、本地图片路径和 `file://` 地址，并补充防盗链请求头、类型及大小校验。
